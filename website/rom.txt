h2. The Ruby Object Model

* Introspects code as written, not the objects produced
* has imperative methods for altering it in place AND transformative methods for producing a copy
  with different formulation
* break away from sexp weirdness
* understands original source line numbers
* transformations preserve line numbers by default
* could this be cross-language????
